# AI Workflows

## Daily Workflow

1. Read `CURRENT_STATE.yaml`.
2. Pick the smallest useful next action.
3. Update the relevant markdown note or YAML index.
4. Add a short `DAILY_LOG.md` entry.
5. Update weak areas or next actions if something changed.

## Weekly Review Workflow

1. Read the last 7 days of `DAILY_LOG.md`.
2. Identify repeated mistakes and skipped topics.
3. Update `CURRENT_STATE.yaml`.
4. Refresh `WEEKLY_PLAN.md`.
5. Move important items into coding, design, or behavioral revisit queues.
6. Add reflection in Chinese and interview actions in English.

## Opportunity Prep Workflow

1. Update `02_opportunities/pipeline.yaml`.
2. Add or refresh the opportunity folder.
3. Map coding, design, and behavioral prep to the role.
4. Identify likely questions.
5. Create a focused 3-day and 7-day review plan if an interview is scheduled.

## Old Repo Import Workflow

1. Copy the old repo into `09_archive/imported_from_Career_Move_2024/`.
2. Do not summarize during the copy.
3. Create an import inventory.
4. Selectively extract high-value notes into active folders.
5. Preserve original context and mark uncertain details as TODO.

## Mock Interview Workflow

1. Add feedback to `07_mock_interviews/feedback_log.md`.
2. Convert weaknesses into action items.
3. Update the relevant index.
4. Schedule revisit dates.
