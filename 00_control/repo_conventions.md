# Repo Conventions

## Naming

- Directory names: English, lowercase, snake_case or numbered prefix.
- File names: English, lowercase, snake_case.
- YAML keys: English, snake_case.
- Markdown headings: bilingual when useful.

## Bilingual Policy

Use Chinese for:

- 中文理解
- Intuition
- Mistake analysis
- Self-reflection
- Raw notes

Use English for:

- Interview scripts
- Final STAR answers
- System design explanation scripts
- Coding explanation scripts
- Commit messages

## YAML Status Values

Preferred values:

- `draft`
- `todo`
- `active`
- `revisit`
- `done`
- `archived`

## Priority Values

- `P0`: interview-tomorrow review item
- `P1`: important active prep
- `P2`: useful later

## Confidence Values

- `low`: cannot explain cleanly yet
- `medium`: can solve or explain with some gaps
- `high`: interview-ready

## Markdown Note Shape

For important prep notes, prefer:

1. 中文理解 / Chinese Understanding
2. English Interview Script
3. Mistakes / 易错点
4. Tradeoffs / 取舍
5. Revisit / 复习状态
