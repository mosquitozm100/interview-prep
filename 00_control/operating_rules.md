# Operating Rules

## Purpose

This repo is an interview preparation operating system, not a note dump.

Every useful session should improve at least one of these:

1. Today's study plan
2. Weak area visibility
3. Interview-tomorrow review readiness

## Daily Rule

Start from `CURRENT_STATE.yaml`.

Pick one small task:

- One coding revisit
- One system design section
- One behavioral story
- One opportunity note

End by updating the relevant index and `DAILY_LOG.md`.

## Weekly Rule

Once per week:

- Review `DAILY_LOG.md`
- Update weak areas in `CURRENT_STATE.yaml`
- Refresh `WEEKLY_PLAN.md`
- Move items in or out of revisit queues
- Choose the next mock interview focus

## File Discipline

- Use YAML files as indexes.
- Use markdown files for thinking, scripts, and templates.
- Do not create files before deciding whether an existing index entry is enough.
- Keep raw notes intact.
- Mark drafts and TODOs clearly.

## Interview Discipline

- Chinese is for understanding, reflection, and debugging thinking.
- English is for the final interview script.
- Avoid fake metrics.
- Prefer concise claims with verified context.
