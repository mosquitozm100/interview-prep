# Weekly Plan

This is a low-pressure restart plan for weekdays with 30-60 minutes and weekends with 2-3 hours.

## Weekly Shape

| Session | Focus | Target |
| --- | --- | --- |
| 1 | Coding | Revisit one P0 problem and write the English explanation |
| 2 | Coding | Practice one current focus pattern |
| 3 | Coding | Timed solve or redo a missed problem |
| 4 | System Design | Review one core design, starting with Rate Limiter or Eval Platform |
| 5 | System Design | Deep dive one weakness, such as observability or async jobs |
| 6 | Behavioral | Refine one STAR story with verified details only |
| 7 | Weekly Review | Update weak areas, revisit queue, and next actions |

## Suggested Week

- Monday: Coding P0 revisit, 30-45 minutes
- Tuesday: System design fundamentals, 30-60 minutes
- Wednesday: Coding pattern refresh, 30-45 minutes
- Thursday: Behavioral story draft, 30-45 minutes
- Friday: Light coding or review, 30 minutes
- Saturday: System design deep dive and English script, 2 hours
- Sunday: Weekly review plus one mock drill, 1-2 hours

## Rule

Consistency beats intensity. A useful 35-minute session is a win.
